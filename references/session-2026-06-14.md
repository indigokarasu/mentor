### Issue: Stray `}` in shell heredoc variable interpolation
Three terminal() calls failed because `${DATA_DIR}}` produced `/root/.hermes/commons/data/mentor/}`. The `${VAR}` syntax inside heredocs can introduce stray braces when the closing `}` is adjacent to other characters.

**Root cause:** When the skill's code examples use `${VAR}` for path construction in heredocs, the shell parser can misinterpret the boundary between the variable name and surrounding text, especially when the variable is followed by `}` from a JSON closing brace or another shell construct.

**Fix:** Use `$VAR` without braces for path interpolation in heredocs. Better: use Python heredocs with hardcoded paths for all backup writes.

### Pattern established
The reliable cron-mode backup write pattern is:
1. `python3 << 'PYEOF'` with hardcoded paths
2. Write to `/tmp/` first
3. Validate with `os.path.getsize()` or `wc -l`
4. Append to persistent log via `open(..., 'a')`

NEVER rely on shell variable interpolation for backup write paths.
